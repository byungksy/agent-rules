# 트럼피아 이력 추가 워크플로우

이 워크플로우는 선택된 코드나 기능을 분석하여 이력서에 사용할 수 있는 형태의 문서로 정리하고, 이를 지정된 이력 관리 폴더에 저장합니다.

## 1. 코드 분석 (Code Analysis)
사용자가 지정한 파일 또는 현재 활성화된 파일을 분석합니다.
- **주요 기능**: 해당 코드가 수행하는 비즈니스 로직 및 기능
- **기술 스택**: 사용된 언어, 라이브러리, 패턴, 아키텍처
- **해결 과제**: 코드에 내포된 기술적 난제 또는 해결된 이슈 (주석, 히스토리 기반 추론)

## 2. 문서 작성 (Documentation)
분석된 내용을 바탕으로 Markdown 문서를 작성합니다. 문서 양식은 다음을 따릅니다:

```markdown
# YYYY.MM.DD - [기능/이슈명]

## 1. 개요
*   **작성일**: [오늘 날짜]
*   **관련 파일**: [분석한 파일 경로]

## 2. 기능 상세 (Features)
*   [기능 1]
*   [기능 2]

## 3. 기술적 특징 (Technical Details)
*   **Language**: [언어]
*   **Framework/Lib**: [라이브러리]
*   **Pattern**: [디자인 패턴]

## 4. 이력서 활용 포인트 (Resume Points)
*   [이력서에 넣기 좋은 문장 1]
*   [이력서에 넣기 좋은 문장 2]
```

## 3. 파일 저장 (Save)
1. 파일명은 `YYYY.MM.DD_[기능명].md` 형식으로 생성합니다. (예: `2017.04.20_Footer_Alert_System.md`)
   - 날짜는 코드 내의 주석이나 커밋 날짜를 참조하되, 불분명할 경우 오늘 날짜 또는 사용자가 지정한 날짜를 사용합니다.
2. 생성된 파일은 **`~/트럼피아 이력/`** 디렉토리에 저장합니다.
3. 저장 후 사용자에게 저장된 경로를 알려줍니다.
